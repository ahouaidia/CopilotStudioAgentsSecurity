Anatomy of a Copilot Studio Agent

Understanding the building blocks of a Copilot Studio agent is the first step toward securing it. Each component introduces unique capabilities — and potential attack surface.

Agent Architecture Overview
Core

Orchestrator

LLM-powered reasoning engine that plans, coordinates actions, and generates responses

Topics & Instructions

System prompts, conversation flows, and behavioral guardrails that define what the agent can and cannot do.

Knowledge Sources

SharePoint, Dataverse, websites, and uploaded files that the agent uses for grounded responses via RAG.

Actions & Connectors

Power Automate flows, custom connectors, and API plugins that let the agent take real-world actions.

Channels

Teams, websites, custom apps — the surfaces where users and other agents interact with the agent.

Authentication

Entra ID integration, SSO, and OAuth — controls who can talk to the agent and what identity it uses to call APIs.

Governance & DLP

Power Platform DLP policies, environment controls, and admin-level guardrails that constrain agent behavior.

Why anatomy matters for security

Each component of a Copilot Studio agent represents a potential attack vector. The orchestrator can be manipulated through prompt injection, knowledge sources can leak sensitive data, actions can be abused for privilege escalation, and channels can be exploited for unauthorized access. Understanding the anatomy is the foundation for building a comprehensive threat model.
