Anatomy of a Copilot Studio Agent

Understanding the building blocks of a Copilot Studio agent is the first step toward securing it. Each component introduces unique capabilities — and potential attack surface.

Copilot Studio Agent Anatomy

Core

Orchestrator

LLM-powered reasoning engine that plans, coordinates, and responds

Topics

Modular conversation flows triggered by user phrases, built from step-by-step nodes.

Attack Surface
  • Prompt injection
  • Unsafe orchestration
  • Embedded secrets

Tools

Connector actions, AI Builder models, and generative answers called at runtime.

Attack Surface
  • Privilege escalation
  • Data exfiltration
  • Governance bypass

Knowledge

Enterprise data from Power Platform, O365, and external sources for grounded answers.

Attack Surface
  • Sensitive info exposure
  • Indirect prompt injection
  • Data oversharing

Identity

Entra ID auth, OAuth, and delegated permissions governing agent and user access.

Attack Surface
  • Token theft & replay
  • Over-privileged permissions
  • Auth bypass

LLM Model

Language model powering reasoning, generation, safety filters, and content moderation.

Attack Surface
  • Jailbreak & guardrail bypass
  • Hallucination exploitation
  • Model poisoning

Why anatomy matters for security

Each component of a Copilot Studio agent represents a potential attack vector. The orchestrator can be manipulated through prompt injection, knowledge sources can leak sensitive data, actions can be abused for privilege escalation, and channels can be exploited for unauthorized access. Understanding the anatomy is the foundation for building a comprehensive threat model.
